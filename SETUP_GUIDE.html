WC 2026

Player Stats App — Complete Setup Guide

Follow these steps in order · Everything done from your iPhone · All services are free

What's in this folder
index.html
The main app. Shows all players. Reads data from Firestore.
WC2026_Players.xlsx
Excel spreadsheet with all 295 players. Edit this to update stats.
import.html
One-time tool. Loads all 295 players into Firestore on first setup.
upload.html
Upload an edited Excel file to update Firestore anytime.
manifest.json + sw.js
Make the app installable as a PWA on any phone.
icon-192.png / 512.png
App icon that appears on the home screen.
Before you start: You need three free accounts. Create them now if you don't have them — github.com · firebase.google.com · vercel.com
1
Upload all files to GitHub
This is where your files are stored online
1
Open Safari → go to github.com → sign in
2
Tap the + button (top right) → New repository
3
Name it wc2026-stats · Set to Public · Tick Add a README file · Tap Create repository
4
Inside your new repo, tap Add file → Upload files
5
Tap choose your files → go to the unzipped folder in Files → select all 8 files (hold select to pick multiple)
6
Scroll down → tap Commit changes
✓ Done when: You can see all 8 files listed in your GitHub repo.
2
Create a Firebase project & Firestore database
This is where your player data lives
1
Open Safari → go to console.firebase.google.com → sign in with Google
2
Tap Add project → name it wc2026-stats → tap Continue
3
Turn off Google Analytics → tap Create project → wait → tap Continue
4
In the left menu tap Firestore Database → tap Create database
5
Choose Start in test mode → tap Next
6
Select a location (choose the one nearest to you) → tap Enable
✓ Done when: You see an empty Firestore database screen.
3
Get your Firebase config keys
You will paste these into 2 files — screenshot this screen
1
In Firebase Console, tap the ⚙️ gear icon (top left) → Project settings
2
Scroll down to Your apps → tap the </> web icon → name it wc2026-web → tap Register app
3
Take a screenshot of the config block that appears. It looks like this:
apiKey: "AIzaSyXXXXXXXXXXXX",
authDomain: "wc2026-stats.firebaseapp.com",
projectId: "wc2026-stats",
storageBucket: "wc2026-stats.appspot.com",
messagingSenderId: "123456789012",
appId: "1:123456789012:web:abc123def456"
Important: Keep this screen open or screenshot it. You need these values in Step 4.
4
Add your config to index.html and upload.html
Do this for both files on GitHub
1
Go to github.com → your wc2026-stats repo → tap index.html
2
Tap the pencil icon to edit
3
Find this section (around line 30) and replace every PASTE_YOUR_... with your real values from Step 3:
Find this:
apiKey: "PASTE_YOUR_API_KEY_HERE",
projectId: "PASTE_YOUR_PROJECT_ID",

Replace with your real values, e.g.:
apiKey: "AIzaSyXXXXXXXXX",
projectId: "wc2026-stats",
4
Tap Commit changes when done
5
Now go back to your repo → tap upload.html → tap the pencil → find and replace the same PASTE_YOUR_... placeholders → tap Commit changes
✓ Done when: Both files updated and committed on GitHub.
5
Deploy with Vercel — get your live URL
Vercel reads your GitHub and puts it live on the internet
1
Open Safari → go to vercel.com → tap Sign up → tap Continue with GitHub → authorise
2
Tap Add New Project
3
You will see wc2026-stats in the list → tap Import
4
Leave everything as default → tap Deploy
5
Wait about 30 seconds → you see a 🎉 screen with your URL
Your live URL will be something like:
https://wc2026-stats.vercel.app
Save this URL — this is what you share with people and what they install on their phone.
6
Import all 295 players into Firestore
Do this once only — takes about 15 seconds
1
In Safari, go to your URL with /import.html at the end:
e.g. https://wc2026-stats.vercel.app/import.html
2
Fill in the three boxes using your values from Step 3:
API Key · Project ID · App ID
3
Tap Import All 295 Players →
4
Watch the progress bar fill up → wait for 🎉 ALL 295 PLAYERS IMPORTED
✓ Done when: You see the success message. Your app now has all players.
7
Lock your database so no one else can change it
Takes 1 minute — important for security
1
Go to console.firebase.google.com → your project → Firestore Database → tap the Rules tab
2
You will see existing rules. Select all the text and replace it with exactly this:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /players/{id} {
      allow read: if true;
      allow write: if false;
    }
  }
}
3
Tap Publish
What this means: Anyone can read (see the players in the app) but nobody can change the data — only you, using the upload tool.

How to update player stats
Edit Excel → upload → done. No code needed ever again.
1
Open Safari → go to your URL with /upload.html:
https://wc2026-stats.vercel.app/upload.html
2
Tap Download WC2026_Players.xlsx at the top of that page to get the latest version of the spreadsheet
3
Open the downloaded file — it opens in Numbers on iPhone automatically
4
Edit any stats. Add a new row to add a player. Delete a row to remove a player. Give new players a unique ID number.
5
In Numbers: tap ··· (top right) → Export → Excel → save to Files
6
Before uploading — go to Firebase Console → Firestore → Rules → change allow write: if false to allow write: if true → tap Publish
7
Back on the upload page — fill in your 3 Firebase config values → tap Tap to choose your Excel file → pick the file → tap Upload & Sync to Firestore →
8
After upload completes → go back to Firebase Console → Rules → change back to allow write: if false → tap Publish
That's it. All users see the updated stats instantly. No redeployment, no GitHub, no code.
